# Stories

## Phase 1 — MVP (shipped)

- As a visitor, I can open the site and see the detected city wall so that I immediately understand the local context.
- As a visitor, I can read recent posts from my city so that I can quickly browse what people nearby are sharing.
- As a visitor, I can write a text-only post up to 120 characters so that posting feels fast and low effort.
- As a visitor, I can see how many characters I have left while typing so that I stay within the limit.
- As a visitor, I am assigned a random pseudonym for my browser session so that I can post anonymously without creating an account.
- As a visitor, I see a small icon or emoji attached to my pseudonym so that the anonymous identity still feels recognizable during my session.
- As a visitor, I can post at most once per minute so that the wall has basic protection against flooding.
- As a visitor, I cannot submit the exact same text repeatedly from the same session so that simple spam is reduced.
- As a visitor, I only see posts that are less than 48 hours old, ordered newest first, so that the wall feels current and temporary.
- As a visitor, I see posts displayed as notes on a shared chalkboard so that the city wall feels distinct from a normal feed.
- As a visitor, I see new posts appear live on the same city wall so that the board feels active while I am viewing it.
- As a visitor, I see older posts fade as they near expiry so that the temporary nature of the wall is visible.
- As a visitor, I see a clear unavailable state when the app cannot determine a supported city so that the product fails predictably.

## Phase 2 — Readability and mobile UX

- As a visitor, I see notes without a glow effect by default so that the board is easy to read over several minutes.
- As a visitor, I see a subtle glow on a note when I hover over it so that focus feels like a reward rather than noise.
- As a visitor, I see enough spacing between notes that the board feels comfortable rather than cramped.
- As a visitor on mobile, I see notes immediately on load so that I can read without scrolling past the composer.
- As a visitor on mobile, the composer is collapsed by default so that the feed gets the full screen.
- As a visitor on mobile, I can tap a floating button to open the composer so that writing is one tap away without it dominating the screen.

## Phase 3 — City Pulse and daily prompts

- As a visitor, I see a City Pulse strip at the top of the wall showing recent activity so that the city feels alive and worth checking.
- As a visitor, I see the note count and an inferred mood for my city in the current hour so that I get a quick read on local energy.
- As a visitor, I see a rotating daily prompt as the composer placeholder so that I always have something to respond to even without a specific thought.
- As a visitor, the prompt and all UI copy appear in my language so that the product feels local regardless of which city I am in.

## Phase 4 — Micro reactions and retention signals

- As a visitor, I can react to a note with a short human response (nod, same, lol, felt that) so that I can acknowledge something without writing a reply.
- As a visitor, I do not need an account to react so that the product stays frictionless.
- As a visitor, I can only react once per note per session so that reactions stay honest.
- As a visitor, I see reaction counts on notes so that popular or resonant notes are discoverable.
- As a returning visitor, I see how many new notes have appeared since my last visit so that I have a reason to look immediately.
- As a visitor who has posted, I see how many reactions my note received so that I feel a small reward for contributing.
- As a visitor, all reaction labels appear in my language so that the tone feels natural in Spanish as well as English.
